<interfaces>
Grouping order is: filter write-only/permanent, split pre-quarantined, stable-sort batch candidates by `(registerType.value,address)`, group only same-type strict adjacency and maximum complete span, read groups, then read pre-quarantined definitions individually in original filtered-input order.

Ordinary `readRegister` never mutates unsupported/permanent/transient state. Only ordered individual fallback does: Code 2 immediately adds unsupported+permanent; other device Modbus errors add permanent on the third occurrence; success clears that transient count; transport errors never increment it.
</interfaces>
</context>

| Boundary                             | Description                                                                     |

| Transport words → decoder            | Malformed or short values must fail before conversion.                          |
| Batch result → public state          | Device errors and suspect values must not be conflated with transport failures. |

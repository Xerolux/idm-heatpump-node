`ModbusTransport` exposes connected state plus connect, close, destroy, and one read operation. Request identity contains unit ID, `RegisterType`, function code 3 or 4, address, count, and optional timeout milliseconds. Returned words are newly owned 16-bit integers. No `modbus-serial`, pymodbus, response object, callback, retry knob, host, or port type crosses this boundary.

Each separate scenario retains the CTR-01 fields `name`, `configuration`, `transport_responses`, `clock`, `operation`, `expected_result`, `expected_requests`, and `expected_state`. Phase-1 `src/contracts/scenario.ts` remains unchanged.
</interfaces>

<threat_model>

## Trust Boundaries

| Boundary                                 | Description                                                                              |
| ---------------------------------------- | ---------------------------------------------------------------------------------------- |
| Caller values → immutable domain objects | Mutable nested options/maps/sets could corrupt shared definitions or cached builds.      |
| Python validation → TypeScript factory   | Additional TypeScript rejection could silently narrow the authoritative public contract. |

## STRIDE Threat Register

| Threat ID | Category               | Component                    | Disposition | Mitigation Plan                                                                                                 |
| --------- | ---------------------- | ---------------------------- | ----------- | --------------------------------------------------------------------------------------------------------------- |
| T-1-01    | Spoofing               | fixture baseline             | mitigate    | Require exact provenance on generated symbol/class/register expectations.                                       |
| T-1-02    | Tampering              | constants/RegisterDef        | mitigate    | Frozen representations, exact pinned acceptance/rejection goldens, cloned nested values, and no invented rules. |
| T-1-03    | Repudiation            | validation outcomes          | mitigate    | Stable documented semantic codes and deterministic accepted/rejected tests identify the exact pinned branch.    |
| T-1-04    | Information disclosure | runtime values/tests         | mitigate    | Synthetic fixtures only; no logging of device values, credentials, PINs, IPs, or identifiers.                   |
| T-1-05    | Denial of service      | domain structures            | accept      | Phase-1 structures are finite repository definitions; no recursive parser or I/O exists in this plan.           |
| T-1-06    | Elevation of privilege | object construction          | mitigate    | Plain own-property cloning/freezing, no eval/prototype merging/I/O, and no caller-controlled derived fields.    |
| T-1-07    | Tampering              | sentinels/accepted domain    | mitigate    | Preserve finite sentinels and Python-accepted inputs exactly; immutability never narrows semantics.             |
| T-1-SC    | Tampering              | package-manager supply chain | accept      | No package installation or dependency change; native language/runtime primitives and locked tests only.         |

<threat_model>

## Trust Boundaries

| Boundary                              | Description                                                                                            |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| Verified checkout → Python import     | Upstream code execution is allowed only after every identity and cleanliness check succeeds.           |
| Python values → language-neutral JSON | Non-finite values, sets, enums, errors, and maps can become lossy or ambiguous during serialization.   |
| Generator → repository filesystem     | A bug or malicious upstream value must not choose a path or partially replace authoritative artifacts. |

## STRIDE Threat Register

| Threat ID | Category               | Component                      | Disposition | Mitigation Plan                                                                                                                                                                                     |
| --------- | ---------------------- | ------------------------------ | ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-1-01    | Spoofing               | upstream checkout/import       | mitigate    | Independently recheck allowlisted remote, clean exact HEAD/tag/version/schema before `sys.path` or import.                                                                                          |
| T-1-02    | Tampering              | generated fixtures             | mitigate    | Pinned executable source, upstream snapshot cross-check, deterministic bytes, committed goldens, atomic writes, and hard check-mode diff.                                                           |
| T-1-03    | Repudiation            | generated evidence             | mitigate    | Embed repository/package/version/tag/full SHA/schema/generator version in every fixture.                                                                                                            |
| T-1-04    | Information disclosure | fixture extraction             | mitigate    | Synthetic semantic inputs only; reject credentials, PINs, private IPs, identifiers, raw captures, and secret-bearing jobs.                                                                          |
| T-1-05    | Denial of service      | recursive contract data        | mitigate    | Closed operation/schema versions, bounded fixture collections/strings, duplicate-name rejection, and no generic arbitrary recursion.                                                                |
| T-1-06    | Elevation of privilege | subprocess/import/output paths | mitigate    | Argument arrays, no shell, fixed executables, canonical checkout, fixed output allowlist, least-privilege CI, and verification before import.                                                       |
| T-1-07    | Tampering              | tagged numbers/errors          | mitigate    | One-key reserved number envelopes, closed tags, documented error codes, and rejection of permissive or mixed objects.                                                                               |
| T-1-SC    | Tampering              | Python dependency supply chain | mitigate    | Use only the pinned upstream-declared `pymodbus` 3.12.1 compatibility dependency in development/CI, with no npm dependency change or runtime inclusion; preserve lockfile and least-privilege jobs. |

</threat_model>

<threat_model>

## Trust Boundaries

| Boundary                                    | Description                                                                                |
| ------------------------------------------- | ------------------------------------------------------------------------------------------ |
| Official pinned schema → native definitions | Manual transcription or cleanup can change protocol identity and later request shapes.     |
| Caller configuration → map builder          | Invalid circuits/zones/rooms or model precedence can include unsafe/unsupported registers. |
| Cached/internal maps → public ReadonlyMap   | Type-only readonly annotations can be cast and mutated at runtime.                         |
| Runtime map → serialized contract           | Sorting/coercion can hide overlaps, sentinels, numeric keys, or missing metadata.          |

## STRIDE Threat Register

| Threat ID | Category               | Component                      | Disposition | Mitigation Plan                                                                                                                   |
| --------- | ---------------------- | ------------------------------ | ----------- | --------------------------------------------------------------------------------------------------------------------------------- |
| T-1-01    | Spoofing               | register fixture authority     | mitigate    | Require exact pinned provenance and upstream-snapshot cross-check from Plan 02.                                                   |
| T-1-02    | Tampering              | definitions/builders/schema    | mitigate    | Central factory, complete 26-field golden equality, exhaustive gates, and positive overlap assertions.                            |
| T-1-03    | Repudiation            | register provenance            | mitigate    | Preserve source/sourceVersion/supportedModels/sentinels/lastVerified in every definition and serialized comparison.               |
| T-1-04    | Information disclosure | register metadata              | mitigate    | Public official metadata and anonymized verification labels only; no raw device captures, PINs, IPs, credentials, or identifiers. |

| T-1-06    | Elevation of privilege | public map mutation            | mitigate    | Fresh/mutation-proof results, deeply frozen definitions, no dynamic code/I/O, and fixed registry indexes.                         |

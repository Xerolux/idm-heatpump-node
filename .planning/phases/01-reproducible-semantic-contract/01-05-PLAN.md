<interfaces>
Primitive `ModbusCodec` mirrors Python's public static surface with camelCase methods: `decodeFloat32`, `encodeFloat32`, `decodeInt16`, `encodeInt16`, `decodeInt8`, and `encodeInt8`. Float32 uses an `ArrayBuffer(4)` and `DataView` with explicit little-endian reads/writes; returned words are low then high unless `swapped` is true.

Input behavior is datatype-specific. Float32 decode mirrors Python `struct.pack("<HH")`, so only its consumed words must be packable unsigned 16-bit integers. INT8/INT16 decoders mirror Python bitwise masking and accept arbitrary integers, including negative and oversized values. Register UCHAR/BITFLAG/BOOL mask their first integer word; unscaled UINT16 returns the first word as pinned. The dispatcher enforces only required array length before the datatype handler.

Register-aware `decodeValue(words, register)` and `encodeValue(value, register)` consume immutable `RegisterDef`, use exhaustive DataType dispatch, and remain pure. They do not send requests, validate model membership, throttle EEPROM, or update cyclic state; those are later phases.

`contracts/api-mapping.json` is the sole authority for the public Node name, owner, and representation of `ModbusCodec`. Codec tests join that row to Python-only class/signature/default/validation facts. Register-aware `decodeValue` and `encodeValue` are internal helpers mirroring the pinned `IdmModbusClient` methods for semantic reuse; they have no mapping rows, are not exported from the package root, and do not mark `IdmModbusClient` complete before Phase 2.

<threat_model>

## Trust Boundaries

| Boundary                                  | Description                                                                                       |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------- |
| Caller numbers/words → binary codec       | JavaScript coercion, narrowing, signedness, or endian defaults can silently alter protocol words. |
| Golden tagged JSON → numeric expectations | NaN/infinity/-0 can be lost if parsed as ordinary JSON numbers.                                   |
| Register metadata → codec dispatch        | Unknown/mutated datatype or contradictory size can route data through the wrong decoder.          |

## STRIDE Threat Register

| Threat ID | Category               | Component                    | Disposition | Mitigation Plan                                                                                                                  |
| --------- | ---------------------- | ---------------------------- | ----------- | -------------------------------------------------------------------------------------------------------------------------------- |
| T-1-01    | Spoofing               | codec fixture provenance     | mitigate    | Require strict fixture baseline parsed by prior contract modules.                                                                |
| T-1-02    | Tampering              | words/results                | mitigate    | Datatype-specific validation/masking, exhaustive immutable dispatch, generated goldens, and explicit endian behavior.            |
| T-1-03    | Repudiation            | codec rejections             | mitigate    | Stable semantic error codes plus named vector cases identify every failure reason.                                               |
| T-1-04    | Information disclosure | codec                        | accept      | Pure numeric transformations contain no secrets, logs, network, or device identifiers.                                           |
| T-1-05    | Denial of service      | input arrays                 | mitigate    | Consume only required bounded words, reject malformed values, and perform no unbounded allocation/recursion.                     |
| T-1-06    | Elevation of privilege | codec implementation         | accept      | Pure DataView/math code performs no I/O, dynamic evaluation, or child process execution.                                         |
| T-1-07    | Tampering              | exceptional numeric states   | mitigate    | Strict tagged expectations, `Object.is`/`Number.isNaN`, Float32 representability checks, and separate primitive/register layers. |
| T-1-SC    | Tampering              | package-manager supply chain | accept      | No package/dependency change; use native DataView and the locked test stack.                                                     |

<threat_model>

## Trust Boundaries

| Boundary                          | Description                                                                                                    |
| --------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| Manifest file → Node parser       | Repository-controlled JSON is still untrusted structured input until every key and value is validated.         |
| Local path → Git checkout         | A caller-supplied directory can point at the wrong repository, a dirty tree, a moved tag, or a malicious path. |
| Manifest/checkout → child process | Values crossing into Git must remain literal arguments and may not enter a shell.                              |

## STRIDE Threat Register

| Threat ID | Category               | Component                             | Disposition | Mitigation Plan                                                                                                                                            |
| --------- | ---------------------- | ------------------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-1-01    | Spoofing               | `UPSTREAM-PARITY.json` and Git origin | mitigate    | Closed manifest schema, allowlisted canonical repository, fixed tag, and full lowercase SHA.                                                               |
| T-1-02    | Tampering              | checkout contents/tag/worktree        | mitigate    | Require clean status, exact HEAD, exact tag target, and pinned `pyproject.toml` identity before success.                                                   |
| T-1-03    | Repudiation            | verifier output                       | mitigate    | Report the full verified URL/version/tag/SHA/schema tuple; later artifacts embed the same provenance.                                                      |
| T-1-04    | Information disclosure | verifier process                      | mitigate    | Do not read environment secrets, fixture captures, PINs, private IPs, or device identifiers; emit baseline metadata only.                                  |
| T-1-05    | Denial of service      | manifest/path input                   | mitigate    | Bound field lengths, require a directory, use synchronous commands with captured bounded diagnostics, and reject unsupported schemas.                      |
| T-1-06    | Elevation of privilege | process invocation/path traversal     | mitigate    | No shell, no command interpolation, canonical caller path, fixed Git subcommands, no upstream-selected executable/output path, and no imports.             |
| T-1-SC    | Tampering              | package-manager supply chain          | accept      | This plan installs no package and uses only already locked Node dependencies plus Git; the Phase-1 package legitimacy audit found no dependency expansion. |

</threat_model>

<threat_model>

## Trust Boundaries

| Boundary                                      | Description                                                                          |
| --------------------------------------------- | ------------------------------------------------------------------------------------ |
| Generated public inventory → editable mapping | A stale or tampered mapping could omit or misroute a public symbol.                  |
| Mapping/manifest → generated Markdown         | Human-readable docs must not become a second authority or conceal incomplete status. |
| Evidence paths → `complete` status            | A string path alone must not falsely prove implementation parity.                    |

## STRIDE Threat Register

| Threat ID | Category               | Component                    | Disposition | Mitigation Plan                                                                                                                            |
| --------- | ---------------------- | ---------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| T-1-01    | Spoofing               | public inventory provenance  | mitigate    | Require full baseline identity and exact generated inventory from Plan 02.                                                                 |
| T-1-02    | Tampering              | mapping/docs                 | mitigate    | Exact one-to-one coverage, strict schema, deterministic generation, and non-mutating hard-fail check mode.                                 |
| T-1-03    | Repudiation            | status/evidence claims       | mitigate    | Record owner, status, normalization, evidence path, and pinned baseline in every generated row.                                            |
| T-1-04    | Information disclosure | documentation                | mitigate    | Mapping/docs contain public symbol metadata only; no device addresses beyond public protocol docs, credentials, PINs, IPs, or identifiers. |
| T-1-05    | Denial of service      | mapping input                | mitigate    | Exactly 89 bounded entries, closed fields/statuses/export paths, and duplicate rejection.                                                  |
| T-1-06    | Elevation of privilege | documentation generator      | mitigate    | Node standard library, fixed paths, no shell/child process/network, atomic allowlisted writes.                                             |
| T-1-SC    | Tampering              | package-manager supply chain | accept      | No dependencies are added or installed by this plan; existing locked Node tooling is used.                                                 |
